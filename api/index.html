    API reference

    Clients

    revolut.RevolutMerchantClient

    Synchronous Revolut Merchant API client.

    Example

    client = RevolutMerchantClient(secret_key="sk_...", environment="sandbox") order = client.orders.create(amount=1000, currency="GBP")

    revolut.AsyncRevolutMerchantClient

    Asynchronous Revolut Merchant API client.

    Example

    async with AsyncRevolutMerchantClient(secret_key="sk_...") as client: ... order = await client.orders.create(amount=1000, currency="GBP")

    Webhooks

    revolut.verify_signature(raw_body, *, signature_header, timestamp_header, signing_secret, tolerance_seconds=DEFAULT_TOLERANCE_SECONDS, now=None)

    Verify a webhook delivery and return its parsed payload.

    Parameters:

    Name Type Description Default
    raw_body str | bytes

    The exact request body as received (do not re-serialize).

    		 required
    signature_header str

    The Revolut-Signature header value.

    		 required
    timestamp_header str

    The Revolut-Request-Timestamp header (Unix ms).

    		 required
    signing_secret str

    The webhook's signing_secret (wsk_...).

    		 required
    tolerance_seconds int

    Reject deliveries older than this (replay protection).

    		 DEFAULT_TOLERANCE_SECONDS
    now float | None

    Current time in epoch seconds; defaults to :func:time.time.

    		 None

    Raises:

    Type Description
    SignatureVerificationError

    If the timestamp is stale or no signature matches.

    revolut.compute_signature(*, timestamp, raw_body, signing_secret)

    Return the expected v1=<hex> signature for a payload.

    Configuration

    revolut.Environment

    Bases: str, Enum

    Target Revolut environment.

    revolut.RetryConfig dataclass

    Controls automatic retries for transient failures.

    backoff_seconds(attempt, retry_after, rng=None)

    Seconds to wait before the next attempt (0-indexed attempt).

    When jitter > 0, up to jitter * base extra seconds are added (using rng, defaulting to :func:random.random) to de-correlate retries from concurrent clients.

    Models

    revolut.Money

    Convenience helper to convert between major and minor currency units.

    The Revolut API always works with integer minor units (e.g. 1099 for £10.99). These helpers make conversions explicit and currency-aware.

    from_major(value, currency) classmethod

    Build from a major-unit value, e.g. Money.from_major("10.99", "GBP").

    to_major()

    Return the major-unit value as a :class:~decimal.Decimal.

    revolut.Order

    Bases: RevolutModel

    A Merchant API order.

    Amounts are integers in the currency's minor units. The public identifier is token; public_id is accepted for compatibility with the legacy create-order endpoint.

    public_token property

    The public identifier, preferring token over legacy public_id.
